# 🌐 Список бесплатных рабочих прокси для Telegram

Обновляется автоматически, проверяется на работоспособность автоматически.

Этот репозиторий — **форк оригинального проекта** [`SoliSpirit/mtproto`](https://github.com/SoliSpirit/mtproto) с упором на удобство использования:

- сохранить оригинальный текстовый формат списка,
- и дать удобный **кликабельный Markdown-список**.

> **Источник (upstream):** `SoliSpirit/mtproto`  
> **Частота синхронизации:** каждые 12 часов

---

## 📦 Что есть в этом форке

В форке поддерживаются **оба формата**:

- [`all_proxies.txt`](all_proxies.txt) — оригинальный «сырой» текст (по одному прокси на строку)
- [`all_proxies.md`](all_proxies.md) — удобный кликабельный список с прямыми fallback-ссылками `tg://proxy`

Выбор по задаче:

- для **скриптов/автоматизации** — `all_proxies.txt`
- для **ручного использования** — `all_proxies.md`

---

## 🔄 Автообновление из оригинального репозитория

Список автоматически синхронизируется из оригинального `SoliSpirit/mtproto` каждые 12 часов.

Автоматизация реализована через:

- `scripts/sync_mtproto.py`
- `.github/workflows/sync-from-solispirit.yml`

> `all_proxies.txt` коммитится только когда его содержимое реально меняется. Если upstream не изменился, GitHub может показывать более старую дату «последнего обновления» этого файла.

## 💡 Как пользоваться

1. Откройте [`all_proxies.md`](all_proxies.md)
2. Нажмите на ссылку сервера, чтобы открыть прокси через `t.me`
3. Если `t.me` заблокирован, скопируйте значение `direct: tg://proxy?...` рядом со ссылкой сервера и откройте его локально

Markdown-список сохраняет `https://t.me/proxy?...` как кликабельную ссылку, потому что GitHub и некоторые desktop-просмотрщики Markdown не всегда разрешают кликабельные нестандартные схемы вроде `tg://`.
Для стран, где `t.me` заблокирован, у каждого прокси также есть копируемый прямой fallback `tg://proxy?...`.

Возможные нюансы:

- ссылки `tg://` требуют установленного Telegram и регистрации Telegram как обработчика таких ссылок в системе;
- некоторые браузеры, встроенные webview, просмотрщики Markdown или интерфейсы Git-хостингов могут спрашивать подтверждение перед открытием Telegram или блокировать нестандартные схемы ссылок;
- если не работает ни кликабельная ссылка `t.me`, ни прямой fallback `tg://`, можно вручную скопировать данные прокси из [`all_proxies.txt`](all_proxies.txt) и добавить их в настройках Telegram.
